# MongoDB数据库Schema设计

## 1. 用户表 (users)
```javascript
{
  _id: ObjectId,
  username: String,          // 用户名（唯一）
  password: String,          // 加密后的密码（bcrypt）
  phone: String,            // 手机号（可选）
  email: String,            // 邮箱（可选）
  role: String,             // 角色：'driver'（货车司机）| 'admin'
  profile: {
    realName: String,       // 真实姓名
    idCard: String,         // 身份证号（加密存储）
    licensePlate: String,   // 车牌号
    vehicleType: String     // 车辆类型
  },
  createdAt: Date,
  updatedAt: Date,
  lastLoginAt: Date
}
```

## 2. 合同表 (contracts)
```javascript
{
  _id: ObjectId,
  userId: ObjectId,         // 关联用户
  fileName: String,         // 原始文件名
  filePath: String,         // 文件存储路径
  fileHash: String,         // 文件SHA256哈希值（用于验证完整性）
  fileSize: Number,        // 文件大小（字节）
  mimeType: String,        // 文件类型
  
  // 合同基本信息（AI解析后填充）
  contractInfo: {
    productName: String,    // 产品名称
    insuranceCompany: String, // 保险公司
    insurancePeriod: String,  // 保险期限
    insuranceAmount: String,  // 保险金额
    isLegitimate: Boolean,    // 是否为正规保险
  },
  
  // AI分析结果
  analysis: {
    riskLevel: String,      // 'low' | 'medium' | 'high'
    clauses: [{
      title: String,        // 条款标题
      original: String,     // 原文
      interpretation: String, // 解读
      risk: String          // 风险等级
    }],
    summary: String,       // 综合分析
    analyzedAt: Date
  },
  
  // 电子存证信息
  evidence: {
    timestamp: Date,        // 上传时间戳
    blockchainHash: String, // 区块链哈希（可选，用于不可篡改）
    signature: String      // 数字签名
  },
  
  status: String,          // 'uploaded' | 'analyzing' | 'completed' | 'failed'
  createdAt: Date,
  updatedAt: Date
}
```

## 3. 证据表 (evidences)
```javascript
{
  _id: ObjectId,
  userId: ObjectId,         // 关联用户
  type: String,            // 'audio' | 'image' | 'document' | 'chat'
  fileName: String,        // 文件名
  filePath: String,        // 存储路径
  fileHash: String,        // SHA256哈希值
  fileSize: Number,
  mimeType: String,
  
  // 证据元数据
  metadata: {
    description: String,    // 描述
    relatedComplaintId: ObjectId, // 关联投诉ID
    relatedContractId: ObjectId,   // 关联合同ID
    location: String,      // 地点（可选）
    recordedAt: Date       // 记录时间
  },
  
  // 电子存证（法律合规）
  timestamp: {
    createdAt: Date,       // 创建时间
    uploadedAt: Date,      // 上传时间
    verifiedAt: Date      // 验证时间
  },
  hash: {
    sha256: String,        // SHA256哈希
    md5: String           // MD5哈希（备用）
  },
  integrity: {
    isVerified: Boolean,   // 是否已验证
    verifiedAt: Date,
    verificationResult: String // 验证结果
  },
  
  status: String,          // 'active' | 'archived' | 'deleted'
  createdAt: Date,
  updatedAt: Date
}
```

## 4. 误导检测报告表 (detection_reports)
```javascript
{
  _id: ObjectId,
  userId: ObjectId,
  sourceType: String,      // 'text' | 'audio' | 'image' | 'document'
  sourceFile: {
    fileName: String,
    filePath: String,
    fileHash: String
  },
  
  // AI分析结果
  analysis: {
    riskLevel: String,     // 'low' | 'medium' | 'high'
    riskScore: Number,     // 风险评分 0-100
    keywords: [{
      word: String,        // 关键词
      risk: String,        // 风险等级
      reason: String,      // 风险原因
      position: Number     // 在文本中的位置
    }],
    suggestions: [String], // 专业建议
    analyzedAt: Date
  },
  
  // 电子存证
  evidence: {
    timestamp: Date,
    hash: String
  },
  
  status: String,          // 'analyzing' | 'completed' | 'failed'
  createdAt: Date,
  updatedAt: Date
}
```

## 5. 投诉表 (complaints)
```javascript
{
  _id: ObjectId,
  userId: ObjectId,
  complaintId: String,     // 投诉编号（唯一，如：C2024112301）
  type: String,            // 'insurance' | 'agent' | 'tongchou'
  title: String,           // 投诉标题
  content: String,         // 投诉内容
  status: String,         // 'submitted' | 'accepted' | 'processing' | 'replied' | 'closed'
  progress: Number,        // 处理进度 0-100
  
  // 关联证据
  evidenceIds: [ObjectId], // 关联的证据ID列表
  
  // 处理记录
  processing: [{
    step: String,          // 处理步骤
    description: String,   // 描述
    operator: String,      // 操作人
    timestamp: Date
  }],
  
  // 回复信息
  reply: {
    content: String,
    repliedAt: Date,
    repliedBy: String
  },
  
  createdAt: Date,
  updatedAt: Date
}
```

## 6. 语音分析表 (voice_analyses)
```javascript
{
  _id: ObjectId,
  userId: ObjectId,
  audioFile: {
    fileName: String,
    filePath: String,
    fileHash: String,
    duration: Number       // 时长（秒）
  },
  
  // 语音识别结果
  transcription: {
    text: String,          // 识别出的文本
    segments: [{           // 分段识别
      text: String,
      startTime: Number,
      endTime: Number
    }],
    dialect: String       // 方言类型
  },
  
  // 误导检测结果
  detection: {
    hasMisleading: Boolean,
    riskLevel: String,
    misleadingSegments: [{
      text: String,
      severity: String,
      reason: String,
      warning: String,
      position: Number
    }],
    analyzedAt: Date
  },
  
  status: String,          // 'uploading' | 'transcribing' | 'analyzing' | 'completed'
  createdAt: Date,
  updatedAt: Date
}
```

## 7. 系统配置表 (system_configs)
```javascript
{
  _id: ObjectId,
  key: String,            // 配置键（唯一）
  value: Mixed,           // 配置值
  description: String,    // 描述
  updatedAt: Date
}
```

## 索引设计
- users: username (unique), phone (unique), email (unique)
- contracts: userId, createdAt
- evidences: userId, type, createdAt
- detection_reports: userId, createdAt
- complaints: userId, complaintId (unique), status, createdAt
- voice_analyses: userId, createdAt

## 数据安全考虑
1. **敏感信息加密**：身份证号、手机号等使用AES-256加密存储
2. **密码安全**：使用bcrypt加密，salt rounds = 10
3. **文件完整性**：所有文件存储SHA256哈希值
4. **时间戳**：使用服务器时间，防止客户端篡改
5. **审计日志**：记录所有关键操作（可选，后续实现）
