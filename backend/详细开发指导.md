## 1. 环境准备

### 1.1 安装Node.js

**macOS (推荐使用Homebrew):**
```bash
# 安装Homebrew（如果还没有）
/bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"

# 安装Node.js
brew install node

# 验证安装
node --version  # 应该显示 v18.x.x 或更高
npm --version   # 应该显示 9.x.x 或更高
```

**Windows:**
1. 访问 https://nodejs.org/
2. 下载LTS版本（推荐18.x或20.x）
3. 运行安装程序，一路下一步
4. 打开命令提示符验证：
```cmd
node --version
npm --version
```

### 1.2 安装MongoDB

**macOS:**
```bash
# 使用Homebrew安装
brew tap mongodb/brew
brew install mongodb-community

# 启动MongoDB服务
brew services start mongodb-community

# 验证是否运行
brew services list
# 应该看到 mongodb-community 状态为 started
```

**Windows:**
1. 访问 https://www.mongodb.com/try/download/community
2. 下载Windows版本
3. 运行安装程序
4. MongoDB会自动作为Windows服务启动

**验证MongoDB:**
```bash
# 连接MongoDB（macOS/Linux）
mongosh

# 如果连接成功，会看到：
# Current Mongosh Log ID: ...
# Connecting to: mongodb://127.0.0.1:27017
# Using MongoDB: 6.x.x
```

### 1.3 安装代码编辑器

推荐使用：
- **VS Code** (免费，功能强大)
- **WebStorm** (付费，专业IDE)

## 3. 配置说明

### 3.1 创建环境变量文件

```bash
# 复制示例文件
cp env.example .env

# 或者手动创建
touch .env
```

### 3.2 编辑.env文件

用文本编辑器打开 `.env` 文件，修改以下配置：

```env
# ========== 必须修改的配置 ==========

# MongoDB连接地址（如果MongoDB在本地，保持默认即可）
MONGODB_URI=mongodb://localhost:27017/traffic_control

# JWT密钥（生产环境必须修改！）
# 生成方法：在Node.js中运行 crypto.randomBytes(32).toString('hex')
JWT_SECRET=your-super-secret-jwt-key-change-in-production-please-use-random-string

# 加密密钥（32字节，64个十六进制字符）
# 生成方法：在Node.js中运行 crypto.randomBytes(32).toString('hex')
ENCRYPTION_KEY=0123456789abcdef0123456789abcdef0123456789abcdef0123456789abcdef

# ========== 可选配置 ==========

# 服务器端口（默认3000）
PORT=3000

# 环境（development开发 / production生产）
NODE_ENV=development

# 文件上传配置
UPLOAD_DIR=./uploads
MAX_FILE_SIZE=10485760  # 10MB
ALLOWED_FILE_TYPES=image/jpeg,image/png,image/jpg,application/pdf,audio/mpeg,audio/wav,audio/m4a

# CORS配置（前端地址）
CORS_ORIGIN=http://localhost:5173
```

### 3.3 生成密钥（重要！）

**方法1：使用Node.js生成**
```bash
# 打开Node.js REPL
node

# 在REPL中运行：
const crypto = require('crypto');
console.log('JWT_SECRET:', crypto.randomBytes(32).toString('hex'));
console.log('ENCRYPTION_KEY:', crypto.randomBytes(32).toString('hex'));

# 复制输出的值到.env文件
# 按 Ctrl+C 退出
```

**方法2：使用在线工具**
- 访问 https://www.random.org/strings/
- 生成64个字符的随机字符串

## 6. 理解代码结构

### 6.1 项目架构（MVC模式）

```
请求流程：
客户端 → 路由(routes) → 中间件(middleware) → 控制器(controllers) → 模型(models) → 数据库
                                                      ↓
                                                   响应返回
```

**用C语言理解：**
```c
// 路由 = 路由表
void route(char *path, void (*handler)(Request *req, Response *res)) {
    // 根据path调用对应的handler
}

// 中间件 = 预处理函数
void middleware(Request *req, Response *res, void (*next)()) {
    // 做一些处理
    next(); // 继续下一个
}

// 控制器 = 业务逻辑函数
void handleLogin(Request *req, Response *res) {
    // 处理登录逻辑
}

// 模型 = 数据结构 + 操作函数
typedef struct {
    char username[50];
    char password[100];
} User;

User* findUser(char *username);
void saveUser(User *user);
```

### 6.2 关键文件说明

#### server.js - 服务器入口
```javascript
// 类似C中的main()函数
const app = express();  // 创建应用（类似初始化）

app.use('/api/auth', authRoutes);  // 注册路由（类似路由表）

app.listen(3000);  // 启动服务器（类似启动服务）
```

#### routes/auth.js - 认证路由
```javascript
// 类似C中的路由表
router.post('/register', validateRegister, register);
//        ↑路径    ↑验证中间件  ↑处理函数
```

#### controllers/authController.js - 认证控制器
```javascript
// 类似C中的处理函数
const login = async (req, res) => {
  // req = 请求（类似函数参数）
  // res = 响应（类似返回值）
  
  const user = await User.findOne({ username });  // 数据库查询
  // 类似C中的：User* user = findUser(username);
  
  res.json({ success: true });  // 返回JSON
  // 类似C中的：return response;
};
```

#### models/User.js - 用户模型
```javascript
// 类似C中的结构体定义
const userSchema = new mongoose.Schema({
  username: String,  // 类似 char username[50];
  password: String  // 类似 char password[100];
});

// 类似C中的结构体操作函数
userSchema.methods.comparePassword = async function(password) {
  // 类似 int comparePassword(User *user, char *password);
};
```

### 6.3 异步编程理解

**C语言（同步）：**
```c
FILE *fp = fopen("file.txt", "r");
char buffer[100];
fread(buffer, 1, 100, fp);  // 等待读取完成
printf("读取完成\n");
```

**Node.js（异步）：**
```javascript
fs.readFile('file.txt', (err, data) => {
  // 这个函数在读取完成后才执行
  console.log('读取完成');
});
console.log('这行会立即执行');  // 不等文件读取
```

**理解：**
- Node.js不会阻塞，文件读取在后台进行
- 读取完成后，回调函数被调用
- 类似C中的信号处理或中断处理

### 6.4 中间件理解

**C语言（函数链）：**
```c
void middleware1(Request *req) {
    // 处理1
}

void middleware2(Request *req) {
    middleware1(req);  // 先执行middleware1
    // 处理2
}

void handler(Request *req) {
    middleware2(req);  // 先执行middleware2
    // 最终处理
}
```

**Node.js（中间件链）：**
```javascript
app.use(middleware1);  // 第一个执行
app.use(middleware2);  // 第二个执行
app.get('/', handler);  // 最后执行

function middleware1(req, res, next) {
  // 处理1
  next();  // 继续下一个
}
```

## 7. 开发新功能

### 7.1 添加新API接口的步骤

**示例：添加"获取用户统计"接口**

#### 步骤1：在控制器中添加函数
```javascript
// controllers/authController.js
const getUserStats = async (req, res) => {
  try {
    // 查询用户的合同数量、证据数量等
    const contractCount = await Contract.countDocuments({ userId: req.user._id });
    const evidenceCount = await Evidence.countDocuments({ userId: req.user._id });
    
    res.json({
      success: true,
      data: {
        contractCount,
        evidenceCount
      }
    });
  } catch (error) {
    res.status(500).json({
      success: false,
      message: '获取统计失败',
      error: error.message
    });
  }
};

module.exports = {
  // ... 其他函数
  getUserStats
};
```

#### 步骤2：在路由中注册
```javascript
// routes/auth.js
const { getUserStats } = require('../controllers/authController');

router.get('/stats', authenticate, getUserStats);
```

#### 步骤3：测试
```bash
curl http://localhost:3000/api/auth/stats \
  -H "Authorization: Bearer $TOKEN"
```

## 8. 常见问题

### 问题1：MongoDB连接失败

**错误信息：**
```
❌ MongoDB连接失败: connect ECONNREFUSED 127.0.0.1:27017
```

**解决方法：**
```bash
# 检查MongoDB是否运行
brew services list  # macOS
# 或
systemctl status mongod  # Linux

# 启动MongoDB
brew services start mongodb-community  # macOS
# 或
sudo systemctl start mongod  # Linux
```

### 问题2：端口被占用

**错误信息：**
```
Error: listen EADDRINUSE: address already in use :::3000
```

**解决方法：**
```bash
# 方法1：修改端口
# 在.env中修改 PORT=3001

# 方法2：关闭占用端口的进程
# macOS/Linux
lsof -ti:3000 | xargs kill -9

# Windows
netstat -ano | findstr :3000
taskkill /PID <PID> /F
```

### 问题3：JWT验证失败

**错误信息：**
```
无效的认证令牌
```

**解决方法：**
1. 检查token是否正确传递：
   ```
   Authorization: Bearer <token>
   ```
2. 检查token是否过期（默认7天）
3. 重新登录获取新token

### 问题4：文件上传失败

**错误信息：**
```
不支持的文件类型
```

**解决方法：**
1. 检查文件类型是否在允许列表中
2. 检查文件大小是否超过限制（默认10MB）
3. 检查 `uploads/` 目录权限

### 问题5：模块找不到

**错误信息：**
```
Cannot find module 'express'
```

**解决方法：**
```bash
# 重新安装依赖
rm -rf node_modules
npm install
```
